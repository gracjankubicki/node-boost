# Next.js 14

This project runs Next 14 with React 18-era App Router semantics. Do not generate APIs introduced for Cache Components or React 19.

- Server `fetch` calls are cached by default unless the request opts out. Control freshness with `cache`, `next.revalidate`, route segment `revalidate`, and cache tags.
- Invalidate cached data with `revalidatePath()` or `revalidateTag()` after writes.
- Do not add the `"use cache"` directive, `cacheLife()`, `updateTag()`, `useActionState`, or `useOptimistic`; they are not Next 14/React 18 defaults.
- `params` and `searchParams` are synchronous in page/layout props. Do not apply the async Next 15/16 signature.
- Match the existing mutation path. Server Actions are available, but an established SWR, route-handler, or client-fetch flow remains valid.
