# Next.js 16

This project runs Next 16. First inspect `next.config.*`: Cache Components is optional and `"use cache"` requires `cacheComponents: true`.

- With Cache Components enabled, opt in at the function/component level with `"use cache"`, control lifetimes with `cacheLife()`, and use tag invalidation supported by that model.
- Without Cache Components, preserve the project's existing `fetch` cache/revalidation or dynamic-rendering approach. Do not insert `"use cache"`, `cacheLife()`, or `updateTag()`.
- Cache Components combines a cacheable shell with dynamic holes defined by `<Suspense>` boundaries. Describe this as active only when `cacheComponents: true` is present.
- `params` and `searchParams` in pages/layouts are async—`await` them.
- When Cache Components is enabled, prefer its APIs for new cache behavior. Otherwise, `fetch(..., { next: { revalidate } })`, route config, or existing wrappers may still be the project's correct model.
- Turbopack is the default bundler; do not add webpack-specific config unless the project already has it.

<code-snippet name="Explicit caching in the data layer" lang="ts">
// Only when next.config.* enables cacheComponents.
export async function getInvoices() {
  "use cache"
  cacheLife("minutes")
  return invoiceListSchema.parse(await apiClient.get("/invoices"))
}
</code-snippet>
