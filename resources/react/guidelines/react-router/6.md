# React Router 6

This project runs React Router 6 through `react-router-dom`.

- Match the established route style: route objects with `createBrowserRouter`, JSX `<Routes>`, loaders/actions, and error elements are all valid v6 patterns.
- Import browser APIs from `react-router-dom`; do not apply React Router 7's merged-package guidance.
- Use `<Navigate>`, `redirect`, `<Link>`, or `useNavigate` for SPA transitions. Full-document navigation remains valid for external URLs, downloads, authentication handoffs, or deliberate reloads.
- Preserve existing loader/query ownership. Do not introduce loaders when the application deliberately keeps server state in TanStack Query, or introduce Query when loaders own it.
- Check new paths against the central route tree for duplicate or shadowed route definitions.
