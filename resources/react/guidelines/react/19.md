# React 19

This project runs React 19. Use its APIs when they match the project's established architecture and configured build capabilities.

- New function components can accept `ref` as a prop. Existing `forwardRef` components remain valid; preserve them when compatibility, a component library, or nearby code expects that API.
- React Compiler is a separate build-time integration. Only rely on automatic memoization after verifying the compiler plugin/configuration is enabled; otherwise preserve intentional `useCallback`/`useMemo`/`React.memo` usage.
- Actions are available: `<form action={fn}>`, `useActionState`, `useFormStatus`, and `useOptimistic`. Adopt them when the app uses an Action-based mutation flow; they do not replace an established query/SWR/form architecture merely because React is 19.
- `use(promise)` unwraps promises in render (with Suspense); `use(Context)` replaces `useContext` and may be called conditionally.
- Metadata tags (`<title>`, `<meta>`) render from components and hoist automatically (in non-Next apps).
- Prefer TypeScript types and default parameters over `propTypes`/function-component `defaultProps` in new TypeScript code.
